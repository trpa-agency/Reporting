Cumulative Accounting Dashboards

Single-file HTML pages for the TRPA annual cumulative-accounting cycle. Organized by the four conceptual tracks.
Active set: 9 dashboards (★ marks the primary view in each track). Source markdown: index.md · Repo: trpa-agency/Reporting

1 · Allocation tracking

Where every residential allocation sits. Three pool states: TRPA pool · Jurisdiction pool · Private development pool.

allocation-tracking.html ★
Audience Staff · daily ops
Sankey + bar + map + AG Grid table; type switcher RES / CFA / TAU; year slider; jurisdiction pills.
Data: data/raw_data/residentialAllocationGridExport.csv (one row per allocation, 2,600 total)
pool-balance-cards.html
Audience Staff · per-pool drilldown
One card per pool: remaining, % drawn, trajectory since 2013.
Data: the analyst’s Additional Development as of April2026.xlsx LT Info Pools Balances sheet
public-allocation-availability.html
Audience Public
Residential allocations remaining in each Tahoe Basin jurisdiction.
Data: same as pool-balance-cards

2 · Source of rights

Where each year’s added units came from. Categories: Pre-1987 / 1987 RP / 2012 RP / Banked / Conversion.

residential-additions-by-source.html ★
Audience Leadership · public
Yearly added residential units by source: Facet (one chart per source) or Combined.
Data: the analyst’s FINAL RES SUMMARY 2012 to 2025.xlsx Summary sheet

3 · Total potential development

Constructed + Banked + Converted + still in pools. The full pipeline.

No standalone dashboard yet - coverage split across #1 and #4.

4 · Total development tracking

How much of everything is actually built. Combines headline caps with live state breakdown.

regional-capacity-dial.html ★
Audience Executive · board
Since-1987 gauges (Residential / RBU / TBU / CFA); capacity utilization bar; 2012 Plan additional 4-card grid.
Data: the analyst’s 2026 PPTX slide 8 + residentialAllocationGridExport.csv

Companion views

Broader development context - not strictly cumulative accounting, but reads against it.

development_history.html
Audience Planners · analysts
Live map of every building footprint by Regional Plan era; year slider 1900–2025; cumulative + per-year charts.
Data: live Tahoe Buildings FeatureServer
development_history_units.html
Audience Housing analysts
Buildings dashboard pivoted to residential units; units split across footprints by sqft. Map filter: All / Residential / Multifamily.
Data: Tahoe Buildings + data/processed_data/buildings_with_units.json
qa-change-rationale.html
Audience the analyst · leadership · QA leads
Per-APN audit log of 2023 / 2026 QA correction campaigns. Searchable by APN, change type, year.
Data: qa_change_events.csv from notebooks/04_load_ca_changes.ipynb
genealogy_solver/
Audience the analyst · QA leads
APN lineage lookup. Walks the genealogy graph for any input (single APN or batch CSV), shows the connected component, and joins 2025 cross-reference inline: residential units, year built, jurisdiction, county.
Data: apn_genealogy_tahoe.csv + PDH_2025_OriginalYrBuilt.csv, pre-joined to html/genealogy_solver/data/genealogy_solver.json

Refresh commands

When the analyst sends a refreshed xlsx, drop it in data/raw_data/ and run the matching converter. All dashboards fetch(..., {cache: 'no-cache'}) so reloads pick up new data immediately.

# Set up alias once
PY="C:/Program Files/ArcGIS/Pro/bin/Python/envs/arcgispro-py3/python.exe"

# Per-allocation grid → CSV (allocation-tracking + regional-capacity-dial cards)
PYTHONIOENCODING=utf-8 "$PY" parcel_development_history_etl/scripts/convert_allocation_grid.py

# Buildings × units join → JSON (development_history_units)
PYTHONIOENCODING=utf-8 "$PY" parcel_development_history_etl/scripts/build_buildings_with_units.py

Tech stack

  • Plotly.js (charts) - load before ArcGIS; Dojo’s AMD loader captures Plotly’s UMD wrapper otherwise
  • ArcGIS Maps SDK 4.31 (maps, where needed)
  • AG Grid Community (sortable / filterable tables)
  • Open Sans + TRPA brand tokens (--trpa-blue #0072CE, --trpa-navy #003B71, --trpa-orange #E87722, --trpa-forest #4A6118)
