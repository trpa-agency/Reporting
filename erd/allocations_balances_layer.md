# AllocationsBalances - new Cumulative_Accounting REST layer

Designed 2026-05-15. Normalization of the analyst's `AllocationsBalances_May2026.xlsx`. Path-to-REST for the per-source x per-commodity summary that allocation-tracking + the Tracker currently approximate from CSV computation + hardcoded values.

## Source

- **File**: `data/from_analyst/AllocationsBalances_May2026.xlsx`
- **Shape**: 3 blocks (Grand Total / 1987 RP / 2012 RP) x 4 commodities (RES / RBU / CFA / TAU) = 12 rows
- **Cadence**: snapshot, regenerated by the analyst on the cumulative-accounting cycle. Replace the xlsx when a refresh arrives and re-run the converter.
- **As-of date**: 2026-05-15 (carried in the row, not implied by load time)

## Identity that the data preserves

For every row:
```
AllocatedToPrivate = TotalAuthorized - (JurisdictionPool + TRPAPool + Unreleased)
TotalBalanceRemaining = JurisdictionPool + TRPAPool + Unreleased
```
All 12 rows pass at 2026-05-15. The converter checks and warns on mismatch.

## Normalized schema (proposed SDE table)

```sql
CREATE TABLE dbo.AllocationsBalances (
    ObjectID               INT IDENTITY(1,1) NOT NULL PRIMARY KEY,
    Source                 VARCHAR(50)  NOT NULL,   -- 'Grand Total' | '1987 Regional Plan' | '2012 Regional Plan'
    Commodity              VARCHAR(50)  NOT NULL,   -- 'Residential allocations' | 'Residential bonus units' | 'Commercial floor area' | 'Tourist accommodation'
    CommodityCode          VARCHAR(4)   NOT NULL,   -- 'RES' | 'RBU' | 'CFA' | 'TAU'
    TotalAuthorized        INT          NOT NULL,
    AllocatedToPrivate     INT          NOT NULL,
    JurisdictionPool       INT          NOT NULL,
    TRPAPool               INT          NOT NULL,
    Unreleased             INT          NOT NULL,
    TotalBalanceRemaining  INT          NOT NULL,
    AsOfDate               DATE         NOT NULL,
    LoadedDate             DATE         NOT NULL,
    SourceFile             VARCHAR(120) NOT NULL,
    CONSTRAINT UQ_AllocationsBalances UNIQUE (Source, CommodityCode, AsOfDate)
);
```

### Why wide instead of long

The metric columns are a fixed, well-known set with a stable arithmetic relationship. Wide rows match the analyst's mental model and the dashboard read pattern (one query, one row, four cards). Long form would force every dashboard to pivot back. If we later need a metric the analyst doesn't yet publish, add a column.

### Why include Grand Total

Grand Total is `1987 + 2012` and could be derived. Keeping the analyst's pre-rolled totals as their own rows (a) avoids client-side sums and (b) lets dashboards distinguish "what the analyst published" from "what the dashboard computed." Three sources, four commodities, 12 rows total — cheap to store, easy to query.

   Emits `data/processed_data/allocations_balances.csv` + `.json`.

2. **Load to SDE** (one-time or refresh-as-needed):
   - Create the empty table per the DDL above on the same Enterprise GDB that backs Cumulative_Accounting
   - Use `arcpy.conversion.ExportTable` (or `da.InsertCursor` for an UPSERT-by-`(Source, CommodityCode, AsOfDate)` pattern)
   - First load: bulk insert all 12 rows from the CSV
   - Refresh load: delete rows matching the new AsOfDate first, then re-insert

3. **Publish to REST**:
   - In ArcGIS Pro, add the table to the Cumulative_Accounting MapServer source mxd/aprx (no spatial join needed; it's a standalone table)
   - Republish the service
   - Resulting endpoint: `https://maps.trpa.org/server/rest/services/Cumulative_Accounting/MapServer/10` (next free layer id)
   - Smoke test:
     ```
     /10/query?where=Source%3D%27Grand+Total%27&outFields=*&f=json
     ```

4. **Update `index.html`** Data Sources table - add a row for layer 10 with "Used by" pointing at the dashboards repointed in step 5.

5. **Repoint dashboards**:
   - **tahoe-development-tracker** sections iii + iv: replace hardcoded values with layer 10 reads where `Source='Grand Total'`. Section iii's "Reserved, not constructed" is the difference between `AllocatedToPrivate` and the PDH-counted "built" - which is actually a different concept; needs analyst confirmation that sections iii + iv are derivable from this table.
   - **allocation-tracking** commodity rows: replace per-status CSV computation with layer 10 reads where `Source='2012 Regional Plan'` (per current view) or `Source='Grand Total'` (per Ken's preferred view).
