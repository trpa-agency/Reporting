# CLAUDE.md - Reporting

Project-specific overlay on `Documents\GitHub\CLAUDE.md`. Slim by design.

## What this repo does

Annual TRPA cumulative accounting cycle artifacts: Parcel Development History ETL, dashboards, and analyst CSVs that feed the §16.8.2 report (RES / TAU / CFA / pool balances).

## Conventions

- **APN canonicalization**: always use `canonical_apn(raw)` from `utils.py`. Never roll your own regex. Pads any standard `NNN-NNN-NN(N)` to 3-digit third segment; handles the 2018 leading-zero reformat across counties.
- **Genealogy**: never read individual `apn_genealogy_master/accela/ltinfo/spatial.csv` directly. Always read the consolidated `apn_genealogy_tahoe.csv`, regenerated by `scripts/build_genealogy_tahoe.py`. Apply with the canonical pattern in `s02b_genealogy.py` (filter `is_primary=1, in_fc_new=1, change_year notna`; sort by source_priority/change_year; conflict-skip).
- **Logging**: `from utils import get_logger; log = get_logger("module")`. Logs to console + `parcel_development_history_etl/logs/etl_YYYYMMDD.log`.
- **QA outputs**: use `utils.write_qa_table(df, gdb_path)` - writes both the GDB table and a CSV mirror in `data/qa_data/`.
- **Field name constants**: never hard-code `"APN"`, `"YEAR"`, etc. Use `FC_APN`, `FC_YEAR`, etc. from `config.py`.
- **Plotly hover convention** (dashboards in `html/`): value + unit in the body; category / series name in `<extra>` (renders as the tooltip title). Use `,` thousands formatting for counts. Examples:
  - Lines / area: `'<b>%{y:,}</b> units<extra>%{x}</extra>'` (year in extra) or `'<b>%{y:,}</b> units<extra>1987 Plan</extra>'` (series in extra)
  - Vertical bars: `'<b>%{y:,}</b> units<extra>%{x}</extra>'`
  - Horizontal bars: `'<b>%{x:,}</b> units<extra>%{y}</extra>'`
  - Percentages: `'<b>%{y:.1f}%</b><extra>%{x}</extra>'`
  - Stacked area with `hovermode: 'x unified'`: put the **series** in `<extra>` so it renders as the per-row label; the unified tooltip header gives the year automatically.
- **Plotly color**: always pull from the TRPA palette JS constants (`TRPA.blue` / `TRPA.navy` / `TRPA.orange` etc.); use the `hexA(hex, alpha)` helper for translucent fills / Sankey link colors - never hardcode `rgba(...)` strings inline.
- **No em-dashes**: never use em-dashes (`—` U+2014 or the `&mdash;` HTML entity) anywhere - not in dashboard copy, not in docs, not in code comments, not in commit messages. Use a hyphen `-` (with surrounding spaces for parenthetical breaks: ` - `), a colon `:`, or restructure into two sentences. Applies to all files in this repo. En-dashes (`–` U+2013) for ranges like `2012-2025` are fine to keep but a hyphen is also acceptable.
- **No name attribution**: never reference TRPA staff by name in dashboards, docs, code comments, or commit messages. Use neutral attribution: "the analyst" for the data-providing analyst, "TRPA leadership" for management, "the agency" for institutional decisions. Applies to all files except `data/permit_data/` (raw permit records may contain customer names) and `_archive/` (frozen historical content). Reason: dashboards and docs are public-facing or shared widely; attribution makes maintenance brittle when staff change and risks calling out individuals in public artifacts.

## Adding a new ad-hoc script

Drop it in `parcel_development_history_etl/scripts/`. Mirror the pattern of `scripts/build_2025_yrbuilt.py` (small/simple) or `scripts/build_2025_layer.py` (multi-step). Always:
- `sys.path.insert(0, str(Path(__file__).resolve().parents[1]))` then `from config import ...`
- Read constants from `config.py`, not hard-coded strings
- Use `canonical_apn` for any APN normalization
- Write outputs under `data/processed_data/` (analyst-facing) or `data/qa_data/` (QA) - not the script directory
