Annual TRPA cumulative accounting cycle artifacts: Parcel Development History ETL, dashboards, and analyst CSVs that feed the §16.8.2 report (RES / TAU / CFA / pool balances).

## Conventions

- **APN canonicalization**: always use `canonical_apn(raw)` from `utils.py`. Never roll your own regex. Pads any standard `NNN-NNN-NN(N)` to 3-digit third segment; handles the 2018 leading-zero reformat across counties.
- **Genealogy**: never read individual `apn_genealogy_master/accela/ltinfo/spatial.csv` directly. Always read the consolidated `apn_genealogy_tahoe.csv`, regenerated by `scripts/build_genealogy_tahoe.py`. Apply with the canonical pattern in `s02b_genealogy.py` (filter `is_primary=1, in_fc_new=1, change_year notna`; sort by source_priority/change_year; conflict-skip).
- **Logging**: `from utils import get_logger; log = get_logger("module")`. Logs to console + `parcel_development_history_etl/logs/etl_YYYYMMDD.log`.
- **QA outputs**: use `utils.write_qa_table(df, gdb_path)` — writes both the GDB table and a CSV mirror in `data/qa_data/`.
- **Field name constants**: never hard-code `"APN"`, `"YEAR"`, etc. Use `FC_APN`, `FC_YEAR`, etc. from `config.py`.

## Genealogy methodology

See `parcel_development_history_etl/GENEALOGY.md` for the four-source merge strategy, the `lost_apn` filter, and how to add new genealogy sources.

## Adding a new ad-hoc script

Drop it in `parcel_development_history_etl/scripts/`. Mirror the pattern of `scripts/build_2025_yrbuilt.py` (small/simple) or `scripts/build_2025_layer.py` (multi-step). Always:
- `sys.path.insert(0, str(Path(__file__).resolve().parents[1]))` then `from config import ...`
- Read constants from `config.py`, not hard-coded strings
- Use `canonical_apn` for any APN normalization
- Write outputs under `data/processed_data/` (analyst-facing) or `data/qa_data/` (QA) — not the script directory
